C# 입문 0-23

partial 클래스는 두 개 이상의 파일로 분할을 해서 작성을 할 수 있는 방법이다. 하지만 컴파일 과정에서는 하나의 파일로 결합된다. 효율적인 코드 관리와 다른 개발자와 분장해서 코딩을 할 수 있다.

클래스명 앞에 partial 을 붙혀주면 된다.

예를들어 Ch_23_1 클래스와 Ch_23_2 클래스 파일에 적용해 보고 싶을 경우

partial class Ch_23_1 이라고 클래스명 앞에 붙혀주고

Ch_23_2 클래스의 앞에도 partial 을 붙혀주는데 이때

partial class Ch_23_2 가 아닌 Ch_23_1이라고 해야한다.(같이 분할하려는 클래스의 이름으로)

partial class Ch_23_1 이라고 이름을 바꿔도 오류가 나지 않는다 (원래 클래스 파일의 이름과 클래스명이 다르면 오류가 나게 된다.) partial 이 사용되었기 때문이다.

이렇게 따로 작성해주고 난 후 컴파일 할 때는 하나의 클래스로 결합되어 실행 되는 것.
